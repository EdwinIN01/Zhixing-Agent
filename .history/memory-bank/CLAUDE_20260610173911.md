# CLAUDE 开发规则（Always 永远生效）
## 🔴 绝对禁止规则（违反即终止）
1. ❌ 禁止生成任何超过300行的单个文件，超过必须拆分为多个职责单一的小文件
2. ❌ 禁止修改、删除任何不在当前实施计划步骤中的文件
3. ❌ 禁止重构现有代码，除非我明确说"请重构这部分代码"
4. ❌ 禁止添加任何不在PRD中的功能，哪怕你觉得"更好"或"更完善"
5. ❌ 禁止生成任何我没有要求的注释、说明文档或示例代码
6. ❌ 禁止编造不存在的API、库或功能
7. ❌ 禁止一次性完成多个步骤，一次只做一件事

## 🟡 必须遵守规则
1. ✅ 写任何代码前，必须完整阅读 `memory-bank` 文件夹中的所有文件
2. ✅ 每完成一个实施计划步骤，必须更新 `memory-bank/progress.md`，记录你做了什么
3. ✅ 每新增一个文件，必须在 `memory-bank/architecture.md` 中添加一行说明：`文件名：该文件的核心作用`
4. ✅ 所有函数必须有明确的输入输出，函数名和变量名要见名知意
5. ✅ 任何时候如果有任何疑问，必须先提问，不要假设我的意图
6. ✅ 在我明确说"验证通过"之前，绝对不要开始下一步
7. ✅ 所有错误必须有明确的提示信息，不能静默失败

## 🟢 通用编码规范
1. 使用2个空格缩进
2. 变量名和函数名使用驼峰命名法（Python使用蛇形命名法）
3. 只写必要的注释，解释"为什么"而不是"做什么"
4. 代码要简洁易读，优先考虑可读性而非性能
5. 每个函数只做一件事，保持函数短小精悍
